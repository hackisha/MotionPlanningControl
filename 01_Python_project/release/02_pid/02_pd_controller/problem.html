PD Controller — 비례 + 미분 제어기 구현

Chapter 2 · PID / 02_pd_controller  |  오차의 변화율을 활용해 응답 속도를 높이고 오버슈트를 줄이는 첫 stateful 제어기

P 제어기에 미분(D) 항을 추가해 응답을 더 빠르고 안정적으로 만드는 PD 제어기를 설계해 보세요. 이전 오차를 내부 상태로 기억하는 첫 번째 stateful 제어기입니다.

1문제 상황

01_p_controller 에서는 P 항만으로 위치를 제어했습니다. P 단독 제어기는 오차가 클수록 강하게 밀어붙이지만, 오차가 줄어드는 속도(변화율)는 신경 쓰지 않습니다. 그 결과 목표에 다가오는 중에도 과하게 밀어 오버슈트가 발생할 수 있습니다.

D 항은 "오차가 얼마나 빠르게 줄고 있는지"를 감지하여, 이미 빠르게 줄고 있다면 제어 입력을 줄여 브레이크 역할을 합니다. P 제어와 달리 직전 오차를 기억해야 미분을 근사할 수 있으므로 내부 상태 prev_error가 필요합니다.

2주어진 환경 (Plant)

시뮬레이션 plant는 plant_pd.py의 Plant 클래스(감쇠 점질량)입니다. 01_p_controller 와 동일한 동역학입니다. 

a  ← (u − damping · v) / m
v  ← v + a · dt
y  ← y + v · dt
  • 기본 파라미터: dt=0.1 s, y0=1.0 m, damping=1.0, m=1.0
  • 테스트 게인: kp=2.0, kd=1.0, 시뮬 30초

3구현 과제

pd_controller.py의 PDController.step() 메소드를 구현하세요. 

class PDController:
    def __init__(self, kp: float, kd: float, dt: float): ...

    def step(self, reference: float, measure: float) -> float:
        """한 스텝의 제어 입력 u 를 반환."""
인자 의미
reference 목표값
measure 현재 측정값
반환 제어 입력 u
  • 내부 상태로 prev_error를 관리합니다. 첫 호출 전에는 None입니다.
  • 첫 호출은 D 항 = 0 — 이전 오차가 없으므로 미분 항을 0으로 처리합니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 폐루프 추적 오차 — y0=1.0, target=0.0, kp=2.0, kd=1.0, 30초 시뮬 후 tail 평균 |오차| < 0.01, peak |오차| < 1.2
참고: tail 임계값이 P 과제(0.05)보다 엄격합니다(0.01) — D 항이 트랜지언트를 더 빨리 잠재워야 하기 때문입니다. P 항만 구현하면 tail 임계값을 통과하지 못합니다.

6힌트

  • 오차 부호는 reference − measure 순서입니다 — P 과제와 동일합니다.
  • 미분 근사: (현재 오차 − 이전 오차) / dt를 사용합니다.
  • 첫 호출: 이전 오차가 없으므로 D 기여를 0으로 두고 P 항만 반환합니다.
  • dt로 나누는 것을 잊으면 D 항의 단위가 달라집니다.

7제약

  • plant_pd.py — 수정 금지 (검증 환경).
  • test_pd_controller.py — 수정 금지 (합격 기준).
  • step()의 시그니처 변경 금지.
  • D 항을 위한 prev_error 외에 적분합 등의 추가 내부 상태는 본 과제 범위 밖입니다.
  • 게인 값은 record_gen.py에서 실험 — test_*.py의 값은 변경 금지.
