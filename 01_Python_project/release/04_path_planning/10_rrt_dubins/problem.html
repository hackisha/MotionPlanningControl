Informed RRT* + Dubins — 최대 곡률 제약 아래 (x, y, yaw) 경로 탐색

Chapter 4 · Path Planning / 10_rrt_dubins  |  차량의 최소 회전 반경을 만족하는 부드러운 경로

09 Informed RRT* 의 단일 tree·단일 loop·event-driven round 갱신 구조는 그대로 쓰고, 두 가지를 바꿉니다 — state 가 (x, y) → (x, y, yaw) 로 늘어나고, 두 node 사이의 edge 가 직선이 아니라 Dubins 곡선(최소 회전 반경 R = 1/κ 제약을 만족하는 전진 전용 최단 경로)이 됩니다. 결과적으로 트리·최종 경로가 차량이 실제로 따라갈 수 있는 부드러운 곡선이 됩니다.
이 과제는 차량 제어가 없습니다. 결과는 Rerun 2D viewer 의 트리 성장·informed 수렴과 최종 Dubins 곡선 경로로만 확인합니다. Dubins 자체(6 word 의 길이 공식)는 헬퍼 dubins.py 에 이미 구현돼 있어 호출만 합니다.

1문제 상황

09 Informed RRT* 의 경로는 (x, y) 직선 segment 의 연결이라, 꺾이는 부분에서 yaw 가 즉시 바뀝니다 — 실제 차량은 그렇게 못 움직입니다(조향각·휠베이스로 결정되는 최소 회전 반경 R 이 있어 yaw 가 호 위에서만 변합니다). 10 은 두 node 사이를 잇는 edge 를 곡률 |κ| ≤ kappa 조건을 만족하는 Dubins 최단 곡선 으로 바꿔, 트리·최종 경로 전체가 차량이 따를 수 있는 부드러운 곡선이 되게 합니다.

2Map 구조 S-커브 슬라럼

  • 60×60 격자, 외곽 border — 09 와 같은 슬라럼 맵 (`map_rrt_dubins.py`).
  • start (10, 30, 0), goal (50, 30, 0) — 둘 다 동쪽(yaw=0) 진행.
  • start→goal 일직선을 막는 세로 벽 2개(통과 구멍이 위·아래로 엇갈림) + 모서리 장식 사각형 4개.
  • 경로는 1번 벽을 위로 2번 벽을 아래로 감아 도는 완만한 S-커브. Dubins 곡률 제약 (|κ| ≤ 1/3, R = 3 m) 아래에서 두 게이트 사이의 좁은 회랑을 부드럽게 누빕니다 — kappa 를 키우면 회전 반경이 너무 커져 못 빠져나갑니다.

3구현 도구

3.1 dubins.py — 곡선 생성 헬퍼 (수정 금지)

Dubins 자체(6 word LSL/RSR/LSR/RSL/RLR/LRL 의 normalized 길이 + 호 적분)는 별도 모듈에 이미 구현돼 있습니다. 알고리즘 본문에서 호출만 하세요.

함수역할
dubins_plan(s1, s2, kappa, ds=0.1) 두 (x, y, yaw) 사이의 Dubins 최단 곡선 — (samples, length) 반환. samples 는 s1 다음 점부터 끝점까지 호장 ds 간격 (x, y, yaw) 시퀀스 (s1 자체는 미포함). 정의 불가하면 ([], inf).
truncate_path(path, ds, max_arc) 호장 ds 간격으로 sampling 된 path 를 max_arc 까지 잘라 반환 — steer 거리 제한용.

3.2 rrt_dubins.py 안의 helper

informed_rrt_star() 와 같은 파일 안에 helper 7개가 미리 정의돼 있습니다. import 없이 본문에서 바로 호출하세요 — 직접 다시 구현하지 마세요.

함수역할
_euclid_xy(a, b)두 State 의 (x, y) Euclidean 거리 (yaw 무시) — nearest·near·informed 타원·goal 도달 판정용.
_path_collision_free(samples, obstacles)Dubins fine sample 들의 정수 격자 cell 이 obstacle 가 아니면 True.
_sample_state(rng, grid_size)[0, grid_size]² × [−π, π) uniform random pose.
_make_dubins_connect(kappa, ds)(s1, s2) → (samples, length) 의 closure — 알고리즘 시작에서 한 번만 만들어 재사용.
_steer_to(s_from, s_to, connect, eta, ds)s_from→s_to Dubins 를 호장 eta 까지 잘라 (steer_samples, steer_len) 반환.

중요 — edge_samples 를 보관해야 합니다. Choose-parent 시 채택한 후보의 Dubins samples 를 edge_samples[new_i] 에 저장해 두고, 경로 복원·시각화는 저장된 samples 만 사용하세요. _reconstruct_path 안에서 dubins_plan 을 재호출하면 floating-point 오차로 다른 word 가 선택돼 Bellman 원리가 깨질 수 있습니다 (특히 잘린 steer 의 endpoint 에서 자주 발생). 저장된 sample 이 트리의 단일 진실의 원천입니다.

디버그 신호 — iteration 별 분석 (선택)

informed_rrt_star() 는 optional 인자 dbg 를 받습니다. 탐색 루프 안에서 dbg 가 주어졌을 때 dbg.add(...) 를 호출하면, 매 iteration 의 내부 값이 record 의 debug_scalars 로 저장됩니다 — goal 거리·reject·tree 크기· 재배선 수·현재 round·현재 최단 비용(best_cost) 같은 값을 남깁니다. 

for it in range(max_iter):
    ...
    if dbg is not None:
        dbg.add(goal_dist=..., rejected=..., tree_size=...,
                rewire_count=..., inform_round=..., best_cost=...)   # 매 iteration 한 줄

iteration 은 round 가 바뀌어도 이어지는 전역 카운터이므로, 디버그 신호도 모든 round 가 하나의 연속 타임라인으로 보입니다 — best_cost 가 round 마다 계단식으로 내려가는 것이 informed 수렴입니다. 좌측 entity 패널에서 /debug/<이름> 을 골라 TimeSeriesView 를 추가하세요.

6합격 기준 pytest

알고리즘 형태(자료 구조·RNG 구현)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. start 일치 — path[0] == start.
  2. goal 근접 — ‖path[-1].xy − goal.xy‖ < goal_range (테스트 기준 2.5).
  3. sample 충돌 없음 — 모든 (x, y, yaw) sample 의 정수 cell 이 obstacle 가 아님.
  4. Dubins 적분 연속성 — 연속 두 sample 의 (x, y) 거리 ≤ 2·ds (호장 step 간격 보장).
  5. 곡률 제약 — 연속 두 sample 의 yaw 변화 |Δyaw| ≤ ds · kappa (Dubins 호의 곡률 한계).
  6. 재현성 — 같은 seed 면 같은 path.

7힌트

  • 09 의 골격을 그대로 이식 — 단일 tree·단일 loop·event-driven round 갱신. 차이는 edge 가 직선 → Dubins 곡선 으로 바뀌고, state 에 yaw 가 붙는 것뿐입니다. tree 자료구조 (nodes·parents·children·cost)에 edge_cost·edge_samples 두 dict 가 더 붙습니다 — 부모→자식의 Dubins 호장과 그 호의 fine sample 을 노드별로 저장.
  • connect 클로저는 한 번만 — 알고리즘 시작에서 connect = _make_dubins_connect(kappa, ds) 로 만들어 두고, 이후 nearest 검증· choose-parent·rewire 에서 같은 closure 를 재사용합니다.
  • Nearest 는 Euclidean (x, y) — Dubins distance 정확도는 포기하고 fast heuristic 으로 가까운 node 를 고른 뒤, 실제 Dubins 연결이 정의 불가하거나 충돌이면 reject 합니다. Dubins 는 비대칭이라 i → j 와 j → i 의 호장이 다르니, choose-parent·rewire 의 각 후보 i 에 대해 connect(nodes[i], new_pt) 를 새로 호출해야 합니다.
  • informed 타원은 Euclidean — Dubins 호장 ≥ Euclidean 거리이므로, c_best (Dubins) 로 정의된 Euclidean 타원은 Dubins 비용 ≤ c_best 인 모든 경로 node 의 valid superset 입니다 (Gammell 2014 의 표준 trick). _sample_in_ellipse 가 (x, y) 를 주면, yaw 는 rng.uniform(-pi, pi) 로 따로 채웁니다.
  • edge_samples 저장 = Bellman 정확성 — collision-check 를 통과한 그 시점의 samples 를 edge_samples[child] 로 보관하고, 경로 복원은 _reconstruct_path 가 그 stored samples 만 잇습니다. dubins_plan 재호출 X — fp precision 으로 다른 word 가 선택될 수 있습니다.
  • 개선 검사 → on_improve — c_best 갱신 순간에 on_improve(it, c_best, _reconstruct_path(...)) 를 호출하면 record_gen 이 그 round 의 informed 타원·채택 경로를 viewer 에 저장합니다.
