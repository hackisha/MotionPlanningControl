Average Filter — 누적 평균으로 잡음 줄이기

Chapter 1 · Filters / 01_average_filter  |  상태(state) 기반 클래스 설계의 첫 걸음

센서가 매 순간 잡음 섞인 값을 내보냅니다. 들어오는 측정값을 누적 평균하여 신호를 안정화하는 필터를 설계해 보세요.

1문제 상황

실제 센서는 항상 잡음을 포함합니다. 예를 들어 거리 센서가 매 시간 스텝마다 "참값 + 잡음" 을 출력할 때, 지금까지 들어온 모든 측정값을 평균하면 잡음이 점점 줄어들어 참값에 가까워집니다.

단, 전체 측정값을 배열로 저장해 두고 매번 다시 합산하는 방식은 데이터가 많아질수록 느려집니다. 내부 상태(표본 수와 현재 평균)만 기억하는 재귀 갱신 방식으로 구현하는 것이 목표입니다.

2주어진 환경

demo.py 는 평균 5, 표준편차 1 인 정규분포 N(5, 1) 에서 200개 표본을 뽑아 필터를 통과시키고 plotly 그래프로 보여줍니다. 구현 전에 실행하면 NotImplementedError 가 발생합니다.

test_average_filter.py 는 두 가지 시나리오로 합격 여부를 검증합니다. 알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 봅니다.

3구현 과제

average_filter.py 의 AverageFilter.step() 메소드를 구현하세요. 클래스 골격과 시그니처는 이미 주어져 있습니다. 

class AverageFilter:
    def __init__(self): ...

    def step(self, x: float) -> float
        """새 측정값 x 를 받아 갱신된 누적 평균을 반환."""
인자/반환의미
x 새로 들어온 측정값
반환 지금까지의 누적 평균 (첫 호출에서는 x 자체)
  • 내부 상태로 표본 수와 현재 평균만 보관 — 전체 입력 배열 저장 금지.
  • 첫 호출(n=1)에서는 x 그대로 반환되어야 합니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

4실행 방법

모든 명령은 git 저장소 루트에서 실행합니다.

구현 파일

데모 — 시각적 확인 (구현 후)

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세만 검증합니다.

  1. 상수 입력 안정성 — step(2.0) 을 100회 반복 호출 시 출력이 2.0 유지 (오차 < 1e-6).
  2. 노이즈 추적 RMS — 상수 truth=5 + N(0, 1) 노이즈 1만 표본, warm-up(100 표본) 이후 RMS 오차 < 0.1.

6힌트

  • 재귀 형태로 누적 평균을 갱신하면 전체 배열 없이 O(1) 로 동작합니다.
  • 일반 갱신식: 다음 평균 = 이전 평균 + (새 측정값 − 이전 평균) / 표본수
  • 표본 수를 갱신하는 시점(분모로 쓰기 전/후)에 주의하세요 — 순서가 틀리면 첫 표본 통과 테스트가 실패합니다.

7제약

  • test_average_filter.py — 수정 금지 (검증 환경).
  • step() 의 시그니처 변경 금지.
  • 전체 입력 배열(list/np.array)을 멤버로 저장하는 구현 금지 — 표본 수와 평균만 상태로 유지.
