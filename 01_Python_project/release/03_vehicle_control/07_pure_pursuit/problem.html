횡방향 제어 — Pure Pursuit (기하학적)

Chapter 3 · Vehicle Control / 07_pure_pursuit  |  lookahead 점을 향해 한 번의 atan 으로 조향각을 결정하는 stateless 제어기

PID 없이 순수 기하학만으로 조향각을 결정하는 Pure Pursuit 제어기를 구현해 보세요. 차량의 wheelbase 와 lookahead 거리를 이용해 한 번의 atan 계산으로 δ 를 구합니다.

1문제 상황

차량은 Y0 = 2 m(도로 밖)에서 출발해 직선 도로(Y = 0, 첫 40 m)로 수렴한 뒤, sine 형태의 곡선 도로로 진입합니다.

  • 제어기는 stateless — 같은 입력이 주어지면 언제나 같은 출력을 냅니다. 내부 상태를 유지하지 않습니다.
  • lookahead 점은 d_lh = lookahead_time × vx 앞 로컬 x 위치에서 다항식으로 구합니다.
  • 06 의 PID+FF 와 같은 시나리오·같은 파이프라인을 사용해 두 방식의 특성을 비교할 수 있습니다.

2주어진 환경

vehicle_lat_pursuit.py의 VehicleLat kinematic bicycle 모델과 lateral_pipeline_pure_pursuit.py의 LateralPipeline이 제공됩니다. 파이프라인은 Global2Local → PolynomialFitting → PolynomialValue → 여러분의 PurePursuit 순서로 연결합니다.

  • 경로: 직선(Y=0, 0~40 m) + sine(Y = 2·(cos((x−40)/14) − 1)).
  • 초기 조건: Y0 = 2 m, vx = 3 m/s, wheelbase L = 4 m.
  • sample 점 수 = 5, 로컬 평가 x = 0.0~4.5 m.

pure_pursuit.py의 PurePursuit.step() 메소드를 구현하세요. 

class PurePursuit:

        """조향각 δ (rad) 를 반환 — stateless."""
인자의미
coeff로컬 프레임 다항식 계수 (degree+1, 1) — 05 의 출력 그대로
vx현재 차량 속도 [m/s]
반환조향각 δ [rad] — 내부 상태 없음

  • 분모 0 방지 상수 ε = 1e-3이 코드에 이미 정의되어 있습니다.

5합격 기준 pytest

  1. 곡선 경로 추적 오차 — L=4, lookahead_time=1.0, vx=3, Y0=2, 30초 pipeline, tail 평균 |lateral err| < 0.4 m, peak < 2.2 m.

6힌트

  • lookahead 거리: d_lh = lookahead_time × vx. lookahead y: y_lh = poly(coeff, d_lh).
  • ε(_EPS = 1e-3)는 직진 시점(y_lh ≈ 0) 분모 0 을 방지합니다. atan2 대신 1-인자 atan으로 충분합니다.
  • lookahead_time이 클수록 부드럽지만 반응이 느립니다 — 1.0초가 합격 기준의 설정값입니다.
  • 내부에 prev_error 같은 상태를 만들지 마세요 — 같은 입력 두 번 호출 → 같은 출력이어야 합니다.
