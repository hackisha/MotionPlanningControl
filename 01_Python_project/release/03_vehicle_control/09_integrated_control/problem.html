종·횡 통합 제어 — Integrated Control

Chapter 3 · Vehicle Control / 09_integrated_control  |  속도 PID + Time-gap + Pure Pursuit 을 하나의 파이프라인으로 통합

01~08 의 모든 구성요소를 하나의 시나리오에서 동시에 동작시키는 통합 문제입니다. 직선→곡선 2차선 도로에서 lane 2 를 추종하며 달리다가, 앞 차량이 침범해 오면 종 제어 모드를 speed → timegap으로 전환해 안전 거리를 유지하는 파이프라인을 완성해 보세요.

1문제 상황

도로 구조: 직선(50 m) → R=200 m 좌커브(100 m) → 접선 직선의 2차선 도로.

  • ego: lane 2(오른쪽) 중앙에서 vx = 10 m/s 출발 — lane 2 를 계속 따라갑니다.
  • target: lane 1(왼쪽) 50 m 앞, vx = 8 m/s 등속 주행.
  • t = 10 s부터 target 이 5초 동안 lane 1 → lane 2(ego 의 lane)로 침범합니다.
  • 침범 감지 시 ego 의 종 모드: speed(vx=10 추종) → timegap(τ·v_ego 간격 유지).
  • 이후 target 이 원 lane 으로 돌아가도 timegap 모드는 유지(latch)됩니다.

2주어진 환경

vehicle_combined.py의 VehicleCombined (종·횡 결합 모델), control_pipeline.py의 Road, EgoState, TargetState, PipelineOutput dataclass 가 제공됩니다. 05 의 frame_transform.py는 sys.path 로 import 합니다.

  • 도로: 직선(50 m) → R=200 m 좌커브(100 m) → 접선 직선. Road 객체가 제공.
  • target 의 침범 동작은 test/record_gen 에서 하드코딩 — 학생이 변경하지 않습니다.
  • ego 초기 위치: lane 2 중앙, vx = 10 m/s.

3구현 과제

다음 세 파일에 걸쳐 4개 구현 지점이 있습니다.

① lateral_controller.py — PurePursuit.step

class PurePursuit:
    def __init__(self, L: float, lookahead_time: float = 1.0): ...
    def step(self, coeff: np.ndarray, vx: float) -> float

07 의 Pure Pursuit 식과 동일합니다 — self-contained 사본입니다.

② longitudinal_controller.py — speed_step, timegap_step

class LongitudinalController:
    def __init__(self, dt, kp_v, kd_v, kp_g, kd_g, tau_gap=1.5): ...
    def speed_step(self, v_des: float, v_ego: float) -> float
    def timegap_step(self, gap: float, v_ego: float, v_target: float) -> float
메소드동작
speed_step속도 PD — 01 과 동일 구조, err = v_des − v_ego
timegap_steptime-gap PD — desired_gap = tau_gap · v_ego, gap_err = gap − desired_gap, rel_v = v_target − v_ego

주의: timegap_step의 gap은 호출자(ControlPipeline)가 ego heading projection 으로 계산해서 넘깁니다 — 컨트롤러 내부에서 직접 계산하지 마세요.

③ control_pipeline.py — LongitudinalDecision.long_mode, ControlPipeline.step

class LongitudinalDecision:
    def long_mode(self, t, ego, target) -> str  # "speed" | "timegap" (latch)

class ControlPipeline:
    def step(self, t, ego, target) -> PipelineOutput

ControlPipeline.step은 4단계 시퀀스입니다:

  1. perception — ego heading projection 으로 글로벌 ref 샘플링 → Global2Local → PolynomialFitting.
  2. lateral — LateralController.step(coeff, vx)로 δ 계산.
  3. decision — LongitudinalDecision.long_mode로 종 모드 결정.
  4. longitudinal — speed 모드: ax = ax_speed. timegap 모드: ax = min(ax_speed, ax_timegap) (가속 capping).
가속 capping: timegap 모드에서도 ax_speed로 상한을 두어야 합니다. gap 이 크면 timegap 식이 가속 명령을 내지만, 침범 중인 target 앞에서 가속하면 안 됩니다. min(ax_speed, ax_timegap)이 더 보수적인(작은) 값을 선택합니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

5합격 기준 pytest

sub-controller 단위 수식 검증은 06/07/08 에 위임 — 09 는 decision latch + 통합 closed-loop 만 봅니다.

  1. LongitudinalDecision latch — target 침범 감지 후 mode 가 timegap으로 latch (이후 target 이 원 lane 으로 돌아가도 유지).
  2. timegap 모드 진입 — 45초 시뮬 후 decision.invaded == True, 모드 이력에 timegap 존재.
  3. lane 2 추종 — 마지막 5초 tail 평균 |Y_ego − lane2| < 0.4 m.
  4. 감속 발생 — vx tail mean < 9.5 m/s (v_des 10 보다 작아짐).
  5. 과감속 없음 — vx tail mean > 7.0 m/s (target vx 8 까지 과도 감속 X).
  6. 충돌 없음 — min gap > 2 m.

6힌트

  • LongitudinalDecision: target.Y − road.y_center(target.X)가 음수면 ego lane 으로 침범한 것 — self.invaded = True로 latch.
  • ControlPipeline perception: ego heading projection으로 샘플링 — x_global = ego.X + cos(ego.Yaw) · sample_xs. 곡선 도로에서도 ref 가 ego 앞에 분포합니다.
  • speed_step은 timegap 모드에서도 항상 평가해야 합니다 — 상태를 유지해야 하고, ax capping 의 기준값이 됩니다.
  • 다른 lateral controller(Stanley, LatPIDFF)로 교체하려면 lateral_controller.py docstring 의 예시를 참조하세요 — 파일을 수정하지 않고 LateralController adapter 로 wrap 하면 됩니다.
