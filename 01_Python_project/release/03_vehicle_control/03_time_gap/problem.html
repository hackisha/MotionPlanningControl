종방향 속도 제어 — 전방 차량 Time-gap 추종

Chapter 3 · Vehicle Control / 03_time_gap  |  자율주행 ACC(Adaptive Cruise Control) 의 핵심 제어기

자율주행 자동차의 종방향 속도 제어를 하려 합니다. 전방에 주행 중인 차량을 Constant Time-gap policy 로 추종하도록 제어기를 설계해 보세요.

1문제 상황

여러분의 차량(ego)은 앞서 달리는 차량(target)을 일정한 시간 간격(time-gap) 을 두고 따라가야 합니다. 거리를 고정값으로 두는 것이 아니라 —

  • 고속에서는 더 멀리, 저속에서는 더 가깝게 — 간격이 ego 속도에 비례합니다.
  • 이것이 실제 양산차 ACC 가 쓰는 표준 정책입니다. ("앞차와 2초 거리 유지" 같은 안내가 이것)

제어기는 매 시간 스텝마다 ego 의 가속도 명령 u 를 출력하고, ego 차량은 그 명령을 받아 움직입니다. 목표는 target 과의 시간 간격을 설정값 time_gap(기본 1.0초) 으로 수렴·유지하는 것입니다.

2플랜트 모델

ego·target 차량은 vehicle_long_tg.py 의 VehicleLong 종방향 점질량 모델입니다. 매 스텝 dynamics: 

x   ← x  + dt·vx + 0.5·dt²·ax
vx  ← vx + dt·ax
ax  ← clip( u − C·vx² − g·sinθ ,  −ax_limit, +ax_limit )
  • target: drag 없음(Ca=0). 가속도는 시나리오가 외부에서 지정.
  • ego: drag 포함(Ca=0.5). 가속도 명령 = 여러분 제어기의 출력 u.
  • ax_limit = ±2 m/s² — 명령이 크더라도 실제 가속도는 포화됩니다.

3구현 과제

time_gap_pid.py 의 TimeGapPID.step() 메소드를 구현하세요. 클래스 골격과 시그니처는 이미 주어져 있습니다. 

class TimeGapPID:
    def __init__(self, kp: float, kd: float, ki: float, dt: float,
                 time_gap: float = 1.0): ...

    def step(self, target_x: float, ego_x: float, ego_vx: float) -> float
        """한 스텝의 가속도 명령 u 를 반환."""
인자 의미
target_x target 차량의 현재 위치 [m]
ego_x ego 차량의 현재 위치 [m]
ego_vx ego 차량의 현재 속도 [m/s] — target_space 계산에 필요
반환 가속도 명령 u — ego 의 VehicleLong.step() 입력
  • 내부 상태로 prev_error, error_sum 을 관리합니다 (PID 표준).
  • 첫 호출은 D 항 = 0 — 직전 오차가 없으므로 미분 항을 계산할 수 없습니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

4검증 시나리오 — 같은 게인이 둘 다 통과해야 함

동일한 제어기·동일한 게인으로 두 시나리오를 모두 만족해야 합니다.

① 정속 target record_gen_stationary.py

target 이 일정 속도로 주행. ego 가 1.0초 간격으로 안정 수렴하는지 확인.

② 기동 target record_gen_maneuvering.py

target 이 가속·감속을 반복(0 → +1.5 → −1.5 → 0 m/s²). 충돌 없이 추종하고, 정속 복귀 후 다시 1.0초 간격으로 재수렴하는지 확인.

주의: 한쪽 시나리오만 통과하는 게인은 합격이 아닙니다. 정속에 맞춰 과하게 키운 게인은 기동 시나리오에서 진동·추월(충돌)을 일으킵니다.

6합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다. 검증 명령은 5절 참조.

  1. time-gap 식 정확도 — step() 가 (x_target − x_ego) − v_ego·time_gap 오차에 기반한 PID 출력을 내는지.
  2. 첫 호출 D=0 — 직전 오차가 없을 때 미분 항을 0 으로 처리.
  3. 정속 target 수렴 — 50초 시뮬 후 시간 간격이 1.0초 근방(±0.2초).
  4. 기동 target 안전 + 재수렴 — 80초 시뮬 동안 충돌 없음(간격 > 0), 정속 복귀 후 ±0.5초 안으로 재수렴.

7힌트

  • target_space = ego_vx · time_gap 을 매 step() 호출 시 다시 계산하세요. 생성자에서 한 번만 계산하면 가속 구간에서 오작동합니다.
  • 기동 시나리오는 정속보다 진동·overshoot 위험이 큽니다 → D 항이 중요합니다.
  • 오차 부호를 점검하세요 — (reference − measure) 순서. 부호가 뒤집히면 ego 가 target 으로 돌진합니다.
  • 초기 조건은 ACC 인계 시점 — ego 와 target 이 이미 같은 속도, 이미 정상 간격 (gap₀ = v_ego·time_gap) 에서 출발합니다. 큰 간격을 좁히는 cold start 는 범위 밖.

8제약

  • vehicle_long_tg.py, test_time_gap.py — 수정 금지 (검증 환경).
  • step() 의 시그니처 변경 금지.
  • 게인 값(kp, kd, ki) 은 record_gen_*.py 에서 실험 — release 기본값은 0 으로 초기화되어 있으니 직접 채워 응답을 확인하세요. test_*.py 의 값은 변경 금지.
