종방향 속도 제어 — Speed PID

Chapter 3 · Vehicle Control / 01_speed_pid  |  실차 drag 가 있는 종방향 모델에서 목표 속도 추종

자율주행 차량의 종방향 속도 제어를 시작합니다. drag 와 경사 외란이 있는 실차 모델에서 PID 로 목표 속도를 추종하는 제어기를 설계해 보세요.

1문제 상황

여러분의 차량(ego)은 정지 상태(vx = 0)에서 출발해 목표 속도 v_ref = 30 m/s까지 가속하고 이를 유지해야 합니다.

  • 차량에는 drag(공기저항)가 작용해 속도가 오를수록 구동력이 소진됩니다.
  • 제어기는 매 스텝마다 가속도 명령 u를 출력하고, 차량은 그 명령으로 움직입니다.
  • I 항(적분)이 있어야 drag 외란을 완전히 보상해 정상상태 오차를 없앨 수 있습니다.

2플랜트 모델

ego 차량은 vehicle_long_speed.py의 VehicleLong 종방향 점질량 모델입니다. 매 스텝 dynamics: 

x   ← x  + dt·vx + 0.5·dt²·ax
vx  ← vx + dt·ax
ax  ← clip( u − C·vx² − g·sinθ ,  −ax_limit, +ax_limit )
  • drag 계수: Ca = 0.5 (속도 제곱에 비례해 가속도를 소진).
  • 경사: grade = 0 (이번 과제는 평지).
  • ax_limit = ±2 m/s² — 명령이 크더라도 실제 가속도는 포화됩니다.
  • 초기 조건: vx0 = 0 (정지 출발).

3구현 과제

speed_pid.py의 SpeedPID.step() 메소드를 구현하세요. 클래스 골격과 시그니처는 이미 주어져 있습니다. 

class SpeedPID:
    def __init__(self, kp: float, kd: float, ki: float, dt: float): ...

    def step(self, reference: float, measure: float) -> float
        """한 스텝의 가속도 명령 u 를 반환."""
인자 의미
reference 목표 속도 [m/s]
measure 현재 ego 속도 vx [m/s]
반환 가속도 명령 u — ego 의 VehicleLong.step() 입력
  • 내부 상태로 prev_error, error_sum을 관리합니다 (PID 표준).
  • 첫 호출은 D 항 = 0 — 직전 오차가 없으므로 미분 항을 계산할 수 없습니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — record_gen.py 의 아래 줄. release 기본값은 0 이니 여기에 값을 채우세요:

    controller = SpeedPID(kp=0.0, kd=0.0, ki=0.0, dt=dt)

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다. 검증 명령은 4절 참조.

  1. 속도 추적 오차 — vx0=0, v_ref=30 m/s, kp=1.0, kd=0.0, ki=0.005, 50초 시뮬, tail 평균 |vx 오차| < 0.5 m/s, peak |vx 오차| < 31 m/s.
참고: I 항(ki=0.005)이 drag 외란을 보상해 tail MAE를 작게 만듭니다. 단순 구현(return 0 등)은 두 임계값 모두 초과로 차단됩니다.

6힌트

  • 02_pid 챕터의 PID 구조와 거의 동일합니다 — 그 구현을 참고하세요.
  • error = reference - measure (속도 차이).
  • drag(C·vx²)가 일정 외란 역할을 합니다 → I 항이 잔류 오차를 보상합니다.
  • 첫 호출 시 prev_error가 None이면 D 항을 0으로 처리하세요.

7제약

  • vehicle_long_speed.py, test_speed_pid.py — 수정 금지 (검증 환경).
  • step()의 시그니처 변경 금지.
  • 게인 값(kp, kd, ki)은 record_gen.py에서 실험 — release 기본값은 0으로 초기화되어 있으니 직접 채워 응답을 확인하세요. test_*.py의 값은 변경 금지.
