PID Tuning — 게인 튜닝 실습

Chapter 2 · PID / 04_pid_tuning  |  알고리즘은 이미 구현되어 있습니다 — 더 어려운 plant에 맞는 PID 게인을 직접 결정하는 실무 튜닝 경험

알고리즘 재구현이 아닙니다. 외란이 있고 액추에이터 권한이 약한 plant에 대해 PID 게인 KP, KD, KI를 직접 결정해 보세요. "데모를 보고 응답을 확인하고 게인을 조절" 사이클 — 실무 컨트롤 튜닝의 축소판입니다.

1문제 상황

이번 과제의 plant는 03_pid_controller보다 명확히 더 어렵습니다:

  • actuation_gain = 0.5 — 제어 입력이 plant에 절반만 전달됩니다 (액추에이터 권한 약함).
  • disturbance = 0.3 — 상수 외란이 작용합니다.

학생의 작업은 tuning.py 파일의 게인 세 값을 채우는 것 하나뿐입니다. PID 알고리즘(pid_controller.py)은 이미 정답이 들어 있습니다.

2주어진 환경 (Plant)

시뮬레이션 plant는 plant_pid_tuning.py의 Plant 클래스입니다. 

a  ← (actuation_gain · u + disturbance − damping · v) / m
v  ← v + a · dt
y  ← y + v · dt
  • actuation_gain = 0.5 (액추에이터 권한 약함)
  • disturbance = 0.3 (상수 외란)
  • 나머지: dt=0.1 s, y0=1.0 m, damping=1.0, m=1.0

3구현 과제

파일: tuning.py — 아래 세 값을 결정합니다. PID 알고리즘(pid_controller.py)은 이미 들어 있습니다. 

KP: float = 0.0   # ← 학생이 결정
KD: float = 0.0
KI: float = 0.0
게인 주요 역할 탐색 권장 범위
KP 응답 속도 / 정상상태 게인 [1, 5]
KD 진동·오버슈트 억제 [1, 3]
KI 외란·정상상태 오차 보정 [0.3, 1]

4실행 방법

모든 명령은 git 저장소 루트에서 실행합니다.

구현(튜닝) 파일

01_Python_project/release/02_pid/04_pid_tuning/tuning.py

테스트 — 합격 기준 검증

uv run pytest 01_Python_project/release/02_pid/04_pid_tuning/ -v

시나리오 실행 → record.json 생성 + Rerun viewer 자동 실행 (튜닝 보조 — 반복 실행 권장)

uv run python 01_Python_project/release/02_pid/04_pid_tuning/record_gen.py

tuning.py의 게인을 채운 뒤 위 명령으로 응답 모양을 확인하고, Rerun viewer의 scalar 패널에서 제어 입력 u가 너무 출렁이는지도 확인하세요. JSON만 만들고 viewer를 띄우지 않으려면 --no-viewer를 붙입니다.

챕터 전체 viewer로 재생

uv run python 01_Python_project/release/02_pid/simulator_pid.py 01_Python_project/release/02_pid/04_pid_tuning/

디버그 신호 — 심화 분석 (선택)

record_gen.py 의 dbg.add(...) 블록에는 debug1/2/3 주석 placeholder 가 준비돼 있습니다. 주석을 풀고 분석하고 싶은 값·식(예: 추적 오차, 내부 상태)을 채우면 그 신호가 record 에 함께 저장됩니다 — 신호 추가·삭제·수정은 이 dbg.add() 호출의 kwarg 한 줄로 끝납니다.

디버그 신호는 viewer 기본 화면에는 나타나지 않습니다 — Rerun viewer 좌측 entity 패널에서 /debug/<이름> 항목을 골라 TimeSeriesView 를 직접 추가하면 시계열로 볼 수 있습니다. 구현이 막힐 때 내부 신호를 들여다보는 디버깅 도구로 활용하세요.

5합격 기준 pytest

게인 부호·크기 제약 없음 — 동작 명세(behavioral spec)만 검증합니다.

  1. 폐루프 추적 오차 — 외란 0.3 + actuation_gain 0.5 plant, 60초 시뮬 후 tail 평균 |오차| < 0.1, peak |오차| < 1.5
  2. 제어 입력 boundedness — 시뮬 동안 max|u| < 50 (발산형 튜닝·극단 게인 차단)
주의: 게인을 0으로 두면 테스트 1번에서 즉시 실패합니다. 정답 게인은 한 가지가 아닙니다 — 합격 기준을 만족하는 어떤 조합이라도 통과합니다.

6힌트

  • 모두 작은 양수에서 시작 (예: KP=KD=KI=0.5) → demo 실행 → 응답 보고 키우기.
  • KP가 크면 응답이 빠르지만 진동 위험이 있습니다.
  • KD가 크면 제어 입력이 출렁입니다 (scalar 패널에서 확인).
  • KI가 크면 적분 windup으로 진동이 생길 수 있습니다.
  • 외란이 있으므로 KI가 반드시 필요합니다 — 0으로 두면 잔류 오차가 남습니다.

7제약

  • pid_controller.py — 수정 금지 (이 과제는 알고리즘 재구현이 아닌 게인 튜닝).
  • plant_pid_tuning.py — 수정 금지 (검증 환경).
  • test_tuning.py — 수정 금지 (합격 기준).
  • 정답 게인 값은 스스로 결정해야 합니다 — demo를 반드시 실행해 응답을 확인하세요.
