횡방향 제어 — Stanley (heading + cross-track 합성)

Chapter 3 · Vehicle Control / 08_stanley  |  현재 위치의 두 오차를 합성하는 DARPA Grand Challenge 우승 방식

Pure Pursuit 과 다른 두 번째 기하학적 횡방향 제어기입니다. 현재 ego 위치에서 경로까지의 heading 오차와 cross-track 오차를 합성해 조향각을 결정하는 Stanley 제어기를 구현해 보세요. stateless 입니다.

1문제 상황

차량은 Y0 = 2 m(도로 밖)에서 출발해 직선 도로(Y = 0, 첫 40 m)로 수렴한 뒤, sine 형태의 곡선 도로로 진입합니다.

  • Pure Pursuit 이 미래의 lookahead 점을 보는 것과 달리, Stanley 는 현재 위치의 두 오차를 봅니다.
  • 두 방식의 특성을 같은 시나리오·같은 파이프라인으로 비교할 수 있습니다.
  • 역시 stateless — 내부에 이전 호출 상태를 저장하지 않습니다.

2주어진 환경

vehicle_lat_stanley.py의 VehicleLat kinematic bicycle 모델과 lateral_pipeline_stanley.py의 LateralPipeline이 제공됩니다. 파이프라인은 Global2Local → PolynomialFitting → PolynomialValue → 여러분의 Stanley 순서로 연결합니다.

  • 경로: 직선(Y=0, 0~40 m) + sine(Y = 2·(cos((x−40)/14) − 1)).
  • 초기 조건: Y0 = 2 m, vx = 3 m/s.
  • Stanley 는 lookahead 점 없이 현재 위치(x=0)의 다항식 값에서 오차를 추출합니다.
  • 다항식 계수: coeff[-1] = 상수항 = y(0) = cross-track 오차, coeff[-2] = 1차 계수 = heading 오차.

  • stateless — 이전 호출 상태 저장 없음.

6힌트

  • 다항식 계수 인덱스를 확인하세요: coeff[-1]이 상수항(= cross-track 오차 e_y), coeff[-2]가 1차 계수(= heading 오차 ψ_e)입니다.
  • epsilon은 vx ≈ 0 일 때 분모 0 을 막습니다 — 분모에 더해야 합니다.
  • Pure Pursuit 과 비교: Stanley 는 lookahead 없이 현재 위치만 봅니다. 저속에서는 Stanley 가 더 정밀하고, 고속 곡선에서는 Pure Pursuit 이 유리할 수 있습니다.
  • 역시 내부에 상태를 만들지 마세요 — stateless 입니다.
