PID Controller — 비례 + 적분 + 미분 제어기 구현

Chapter 2 · PID / 03_pid_controller  |  외란이 있는 환경에서 정상상태 오차를 0으로 수렴시키는 완전한 PID 제어기

외란(disturbance)이 존재하는 환경에서도 정상상태 오차를 완전히 제거하는 PID(비례 + 적분 + 미분) 제어기를 설계해 보세요.

1문제 상황

PD 제어기는 외란이 없는 환경에서 잘 동작하지만, 일정한 외란(힘 bias)이 가해지면 정상상태에서 오차가 남습니다. 외란이 있을 때 PD만으로는 평형 위치가 목표값에서 어긋난 채 유지됩니다.

I(적분) 항은 오차가 누적될수록 제어 입력을 키워, 결국 외란을 정확히 상쇄하는 입력을 자동으로 생성합니다. ki = 0으로 두면 PD 제어기와 동등 — 하위 호환성을 확인할 수 있습니다.

2주어진 환경 (Plant)

시뮬레이션 plant는 plant_pid.py의 Plant 클래스입니다. 이전 과제 대비 상수 외란이 추가된 버전입니다. 

a  ← (u + disturbance − damping · v) / m
v  ← v + a · dt
y  ← y + v · dt
  • 기본 파라미터: dt=0.1 s, y0=1.0 m, damping=1.0, m=1.0
  • 테스트 외란: disturbance=0.5
  • 테스트 게인: kp=2.0, kd=1.0, ki=0.5, 시뮬 60초

3구현 과제

pid_controller.py의 PIDController.step() 메소드를 구현하세요. 

class PIDController:
    def __init__(self, kp: float, kd: float, ki: float, dt: float): ...

    def step(self, reference: float, measure: float) -> float:
        """한 스텝의 제어 입력 u 를 반환."""
인자 의미
reference 목표값
measure 현재 측정값
반환 제어 입력 u
  • 내부 상태: prev_error (D 항용), error_sum (I 항용 누적합)
  • 첫 호출은 D 항 = 0 — PD 과제와 동일한 정책
  • 누적합은 error_sum += 오차 × dt 형태로 관리합니다

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 외란 하 폐루프 추적 오차 — 외란 0.5, kp=2.0, kd=1.0, ki=0.5, 60초 시뮬 후 tail 평균 |오차| < 0.05, peak |오차| < 1.5
참고: I 항 없이 PD만 구현하면 외란 0.5 하에서 잔류 오차가 약 0.25 남아 tail 임계값(0.05)을 초과합니다. error_sum += 오차 × dt에서 dt 곱을 빠뜨려도 적분 단위가 달라져 실패합니다.

6힌트

  • I 항 누적합: error_sum += 오차 × dt — dt 곱 누락이 자주 발생합니다.
  • D 항 처리는 PD 과제와 동일 (첫 호출 0, 이후 미분 정규화).
  • 외란이 있을 때 PD 단독으로 잔류 오차가 생기는 이유를 먼저 demo로 확인해 보면 I 항의 역할이 명확해집니다.
  • ki=0으로 설정하면 PD 제어기와 동일하게 동작해야 합니다 — 구현 후 확인해보세요.
