PID + Low-Pass Filter — 노이즈 측정 환경에서의 제어 루프 결합

Chapter 2 · PID / 05_pid_with_lpf  |  LPF로 노이즈를 평활한 뒤 PID에 넣는 닫힌 루프 결합 함수 작성 — 액추에이터 보호가 핵심

노이즈가 섞인 측정값을 Low-Pass Filter(LPF)로 평활한 뒤 PID에 전달하는 닫힌 루프 결합 함수를 작성해 보세요. LPF와 PID 알고리즘은 이미 구현되어 있습니다 — 이번엔 두 모듈을 올바른 순서로 연결하는 결합 패턴을 학습합니다.

1문제 상황

실제 센서에는 측정 노이즈가 항상 존재합니다. 노이즈가 섞인 측정값을 PID에 그대로 전달하면, D 항이 노이즈의 빠른 변동을 고주파 신호로 증폭해 제어 입력이 격렬히 출렁입니다 — 액추에이터 마모와 에너지 낭비의 원인이 됩니다.

이 과제의 핵심 학습 포인트는 "추적 정확도가 좋아지는 것"이 아닙니다. LPF를 거치면 제어 입력의 출렁임이 한 자리수 이상 감소합니다 — 이것이 액추에이터 보호의 실제 가치입니다.

2주어진 환경 (Plant)

시뮬레이션 plant는 plant_lpf.py의 Plant 클래스입니다. step(u)는 동역학 갱신 후 true y를 반환하고, measure()는 노이즈가 추가된 측정값을 반환합니다.

  • disturbance = 0.1, measurement_noise_std = 0.25, 노이즈 시드 42
  • LPF: alpha=0.9 (고정), PID: kp=2.0, kd=2.0, ki=0.5
  • 시뮬: 60초

3구현 과제

closed_loop_lpf.py의 closed_loop_step() 함수를 구현하세요. LPF(low_pass_filter.py)와 PID(pid_controller.py)는 수정하지 않습니다. 

def closed_loop_step(
    plant, estimator, controller, target: float,
) -> tuple[float, float, float, float]:
    """한 스텝: measure → filter → control → actuate.
    반환: (y_true, y_measure, y_estimate, u)"""
인자 의미
plant plant_lpf.Plant 인스턴스
estimator LowPassFilter 인스턴스
controller PIDController 인스턴스
target 목표값
반환 (y_true, y_measure, y_estimate, u) — 모두 float

호출 순서 (의도):

  1. y_measure = plant.measure() — 노이즈 포함 관측
  2. y_estimate = estimator.step(y_measure) — LPF 평활
  3. u = controller.step(target, y_estimate) — PID는 추정값 사용 (raw 노이즈 직접 전달 금지)
  4. plant.step(u) — 동역학 한 스텝

5합격 기준 pytest

결합 함수의 내부 구현 형태는 자유 — 인터페이스 + 동작 명세(behavioral spec)만 검증합니다.

  1. 필터 폐루프 추적 오차 — 외란 0.1, 노이즈 std 0.25, seed 42, 60초 시뮬 후 tail 평균 |y_true| < 0.15, peak < 1.5
  2. LPF가 raw 대비 actuator 부드러움 우수 — 동일 plant에 raw 측정값을 PID에 직접 넣은 베이스라인 대비 제어 입력 std가 1/5 이하 (5배 이상 부드러워야 합격)
핵심: 기준 2는 추적 오차가 아닌 제어 입력의 분산을 봅니다. LPF를 우회하거나 약하게 적용하면 std ratio가 5× 미만이 되어 실패합니다.

6힌트

  • 4단계 호출 순서를 정확히 — estimator 갱신이 controller 앞서야 추정값으로 제어 가능합니다.
  • 반환값의 y_true는 plant.y 속성 (true 위치) — plant.step(u)의 반환값과 같습니다.
  • plant.step(u)의 반환값을 measurement로 오해하지 마세요 — 그건 noise 없는 true y입니다.
