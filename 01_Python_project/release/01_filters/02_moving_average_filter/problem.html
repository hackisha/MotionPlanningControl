Moving Average Filter — 최근 N개만 평균하기

Chapter 1 · Filters / 02_moving_average_filter  |  슬라이딩 윈도우로 오래된 측정을 잊는 필터

센서 값이 시간에 따라 변할 때, 전체 누적 평균은 오래된 측정의 영향을 영원히 받습니다. 최근 N개의 측정값만 평균하는 슬라이딩 윈도우 필터를 설계해 보세요.

1문제 상황

누적 평균(Average Filter) 은 표본이 쌓일수록 잡음이 줄어드는 장점이 있지만, 신호가 실제로 변했을 때 오래된 과거 측정값의 영향 때문에 새 값에 느리게 반응합니다.

Moving Average Filter 는 최근 window개의 측정값만 보관하고 그것의 평균을 출력합니다. 윈도우 밖으로 밀려난 오래된 값은 자동으로 잊혀집니다. window 가 클수록 더 부드럽고, 작을수록 변화에 민감하게 반응합니다.

2주어진 환경

__init__ 은 이미 작성되어 있습니다. self.buffer(deque, maxlen=window)가 준비되며, append 시 자동으로 오래된 값이 밀려납니다.

demo.py 는 N(5, 1) 노이즈 200 표본을 슬라이딩 윈도우로 평활화한 결과를 plotly 그래프로 보여줍니다. demo.py 의 window 값은 0 으로 초기화되어 있으니 직접 채워서 실행하세요.

3구현 과제

moving_average_filter.py 의 MovingAverageFilter.step() 메소드를 구현하세요. 

class MovingAverageFilter:
    def __init__(self, window: int = 10): ...

    def step(self, x: float) -> float
        """새 측정값 x 를 추가하고, 현재 보관 중인 표본들의 평균을 반환."""
인자/반환의미
x 새로 들어온 측정값
window 평균에 사용할 최근 표본 수 (__init__ 인자)
반환 현재까지 보관된 표본들의 평균 (윈도우가 덜 찼을 때는 들어온 만큼만 평균)
  • 윈도우가 부분적으로 채워진 구간(n < window)에서도 첫 호출부터 정상 동작해야 합니다.
  • self.buffer의 append → 자동 밀어냄 → 현재 buffer 평균 반환 순서를 활용하세요.

4실행 방법

모든 명령은 git 저장소 루트에서 실행합니다.

구현 파일

01_Python_project/release/01_filters/02_moving_average_filter/moving_average_filter.py

테스트 — 합격 기준 검증

uv run pytest 01_Python_project/release/01_filters/02_moving_average_filter/ -v

데모 — 시각적 확인 (구현 후)

uv run python 01_Python_project/release/01_filters/02_moving_average_filter/demo.py

demo.py 의 window 기본값은 0 입니다. 직접 원하는 값(예: 15)으로 바꿔 실행하며 평활 강도 변화를 비교하세요. test_*.py의 값은 변경 금지입니다.

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세만 검증합니다.

  1. 상수 입력 안정성 — step(2.0) 을 50회 반복 시 출력이 2.0 유지 (오차 < 1e-6).
  2. 노이즈 추적 RMS — window=20, 상수 truth=5 + N(0, 1) 노이즈 1만 표본, warm-up 이후 RMS 오차 < 0.4.

6힌트

  • self.buffer 는 deque(maxlen=window) — append(x) 하면 오래된 값이 자동 제거됩니다.
  • 단순 합산 형태: sum(self.buffer) / len(self.buffer) 로도 합격 기준을 통과할 수 있습니다.
  • 효율적 형태(선택): buffer 가 가득 찬 이후에는 새 값과 밀려난 값의 차이만 더하고 빼는 방식으로 O(1) 갱신이 가능합니다.

7제약

  • test_moving_average_filter.py — 수정 금지 (검증 환경).
  • step() 의 시그니처 변경 금지.
  • demo.py 의 window 값은 자유롭게 변경 가능 — test_*.py 의 값은 변경 금지.
