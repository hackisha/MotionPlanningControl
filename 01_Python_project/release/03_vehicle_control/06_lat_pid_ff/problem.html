횡방향 제어 — PID + 곡률 Feedforward (LatPIDFF)

Chapter 3 · Vehicle Control / 06_lat_pid_ff  |  직선→곡선 경로를 lookahead PID + 곡률 FF 로 추종

직선 구간과 곡선 구간이 이어진 경로를 lookahead PID와 곡률 피드포워드(FF)를 결합해 추종하는 횡방향 제어기를 설계해 보세요. kff = 0이면 순수 PID, kff > 0이면 FF 항이 추가됩니다.

1문제 상황

차량은 Y0 = 1 m(도로 밖)에서 출발합니다. 먼저 직선 도로(Y = 0, 첫 40 m)로 수렴한 뒤, sine 형태의 곡선 도로로 진입합니다.

  • 저속(vx = 3 m/s): 곡률이 작아 PID-only(kff=0)로도 추종 가능.
  • 고속(vx = 10 m/s): 곡률 변화가 빠르게 오므로 FF 항이 오차를 미리 보정해야 성능이 좋아집니다.

두 시나리오를 같은 제어기로 실행해 FF 의 효과를 직접 비교하세요.

2주어진 환경

vehicle_lat_pid.py의 VehicleLat kinematic bicycle 모델과 lateral_pipeline_pid_ff.py의 LateralPipeline이 제공됩니다. 파이프라인은 05 의 Global2Local → PolynomialFitting → PolynomialValue → 여러분의 LatPIDFF 순서로 연결합니다.

  • 경로: 직선(Y=0, 0~40 m) + sine(Y = 2·(cos((x−40)/14) − 1), R≈100 m).
  • 초기 조건: Y0 = 1 m (도로 밖 시작), Yaw0 = 0.
  • sample 점 수 = 5, 다항식 degree = 3.

3구현 과제

lat_pid_ff.py의 LatPIDFF.step() 메소드를 구현하세요. 

class LatPIDFF:
    def __init__(self, kp: float, kd: float, ki: float, kff: float,
                 dt: float, lookahead_time: float = 0.5): ...

    def step(self, coeff: np.ndarray, vx: float) -> float
        """조향각 δ (rad) 를 반환."""
인자의미
coeff로컬 프레임 다항식 계수 (degree+1, 1) column — 05 의 출력 그대로
vx현재 차량 속도 [m/s] — lookahead 거리와 FF 항 계산에 사용
반환조향각 δ [rad]
  • 첫 호출은 D 항 = 0.
  • kff = 0이면 순수 PID 동작 (FF 항 없음).

4검증 시나리오

① 저속 (vx = 3 m/s) record_gen_low_speed.py

직선 + sine 경로, 30초. PID-only(kff=0)로도 추종 가능합니다.

② 고속 (vx = 10 m/s) record_gen_high_speed.py

같은 경로, 15초. 고속에서는 FF 항(kff > 0) 유무에 따라 오차 차이가 분명히 보입니다.

주의: 합격 기준은 저속 PID-only 추종 정확도와, 고속에서 FF 가 PID-only 대비 15% 이상 개선됨을 동시에 요구합니다. kff 를 0 으로만 두거나 너무 크게 두면 한 쪽을 실패합니다.

6합격 기준 pytest

  1. 저속 추종 정확도 — vx=3, kff=0, 30초 pipeline, tail 평균 |lateral err| < 0.3 m, peak < 1.2 m.
  2. 고속 FF 효과 — vx=10, kff=0.1의 평균 오차가 kff=0 대비 15% 이상 감소 — FF 무효 구현을 차단합니다.

7힌트

  • lookahead 거리: d_lh = lookahead_time × vx. error 는 그 거리의 로컬 y 값입니다.
  • FF 항은 다항식 2차 계수에서 나옵니다: coeff[-3]이 2차 계수입니다 (coeff[-1]: 상수, coeff[-2]: 1차).
  • D 항과 FF 항 모두 고속 곡선에서 중요합니다 — 어느 하나만 있으면 나머지 합격 기준을 통과하기 어렵습니다.
