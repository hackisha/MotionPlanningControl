Kalman Filter — 모델과 측정을 결합한 1차원 추정기

Chapter 1 · Filters / 04_kalman_filter  |  Predict → Update 두 단계로 불확실성을 줄이는 상태 추정

단순 평활 필터를 넘어, 시스템 모델과 측정을 결합해 상태를 추정하는 칼만 필터(scalar 1D) 를 구현해 보세요.

1문제 상황

Average / Moving Average / LPF 는 측정값을 평균하거나 평활화하는 데 그쳤습니다. 칼만 필터는 여기서 한 발 더 나아가 시스템이 어떻게 움직이는지 기술하는 동역학 모델(Predict 단계) 과, 들어오는 측정이 얼마나 신뢰할 수 있는지를 나타내는 측정 노이즈(Update 단계) 를 결합합니다.

매 스텝마다 두 단계를 반복합니다.

  • Predict: 시스템 모델로 다음 상태와 불확실성을 예측합니다.
  • Update: 실제 측정값과의 차이(innovation)로 예측을 보정하고 불확실성을 줄입니다.

2주어진 환경

__init__ 은 이미 작성되어 있습니다. 시스템 행렬 A, B, C 와 노이즈 파라미터 Q, R, 초기 추정 분산 P = p0, 초기 상태 x = 0 이 자동으로 설정됩니다.

demo.py 와 tuning_demo.py 의 q, r, p0 기본값은 0 입니다 — 직접 값을 채운 뒤 실행하며 추정 결과를 비교하세요. test_*.py 의 파라미터는 변경 금지입니다.

3구현 과제

kalman_filter.py 의 KalmanFilter.step() 메소드를 구현하세요. Predict 2줄, Update 3줄, return 1줄 — 총 6줄입니다. 

class KalmanFilter:
    def __init__(self, m: float = 1.0, dt: float = 0.01,
                 q: float = 0.1, r: float = 0.9, p0: float = 10.0): ...

    def step(self, measurement: float, control_input: float) -> float
        """한 번의 Predict + Update 를 수행하고 갱신된 추정값을 반환."""
인자/반환의미
measurement 현재 스텝의 측정값 z
control_input 현재 스텝의 외부 제어 입력 u
q 모델 불확실성(process noise variance) — 클수록 측정을 더 신뢰
r 측정 노이즈 분산 — 클수록 모델을 더 신뢰
반환 갱신된 상태 추정값 self.x
  • Predict 결과를 임시 변수(x_pred, p_pred)에 저장한 뒤 Update 단계에서 사용하세요.
  • Predict 결과를 곧장 self.x 에 쓰면 Update 단계가 망가집니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — demo.py 의 아래 줄. release 기본값은 0 이니 여기에 값을 채우세요:

    kf = KalmanFilter(m=1.0, dt=0.01, q=0.0, r=0.0, p0=0.0)

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세만 검증합니다.

  1. 무노이즈 + feedforward 안정성 — q=0.01, r=1.0 설정, 2000 스텝 후 추정값이 truth(5.0) 근방 (오차 < 0.1).
  2. 노이즈 추적 RMS — 같은 setup 에 N(0, 1) 노이즈 측정 1만 스텝, warm-up 이후 RMS 오차 < 0.3.

6힌트

  • Predict 와 Update 두 단계를 분리해서 구현하세요 — Predict 결과를 임시 변수에 담고 Update 에서 사용합니다.
  • 임시 변수 x_pred, p_pred, k 를 만들어 가독성을 확보하세요.
  • Update 의 (1 - K·C) 부호를 주의하세요 — 잘못 쓰면 P 가 발산합니다.
  • 구현해야 할 6줄 알고리즘은 TODO 주석에 그대로 제시되어 있습니다 — 코드로 정확히 옮기는 연습이 핵심입니다.
