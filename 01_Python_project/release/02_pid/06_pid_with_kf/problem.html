PID + 2D Kalman Filter — 모델 기반 추정기와 제어기 결합

Chapter 2 · PID / 06_pid_with_kf  |  LPF 대신 Kalman Filter로 위치를 추정 — 직전 제어 입력(prev_u)을 KF에 정확히 전달하는 것이 핵심

LPF 대신 2D Kalman Filter(KF)를 사용해 노이즈 측정에서 위치를 추정하고, 그 추정값으로 PID를 구동하는 닫힌 루프 결합 함수를 작성해 보세요. KF와 PID 알고리즘은 이미 구현되어 있습니다 — 이번엔 prev_u를 KF에 정확히 전달하는 결합 패턴이 핵심입니다.

1문제 상황

05_pid_with_lpf에서는 LPF로 노이즈를 평활했습니다. LPF는 모델 정보 없이 단순 가중 평균으로 평활하므로 위상 지연이 생깁니다.

2D Kalman Filter는 plant 모델과 직전 제어 입력을 함께 사용해 다음 측정 전에 상태를 예측한 뒤 측정값으로 보정합니다. 이 예측 단계 덕분에 위상 지연이 줄고 추정이 더 정확해집니다.

LPF와의 핵심 시그니처 차이: KF의 step(measurement, control_input)은 측정값 외에 직전 제어 입력 prev_u도 인자로 받습니다. driver loop에서 매 스텝 prev_u를 갱신해야 합니다.

2주어진 환경 (Plant)

시뮬레이션 plant는 plant_kf.py의 Plant 클래스입니다 — 05와 동일합니다. KF 모델 행렬(A, B, C, Q, R)은 테스트/데모에서 fixture로 제공됩니다 — 학생이 결정할 필요 없습니다.

  • disturbance = 0.1, measurement_noise_std = 0.25, 노이즈 시드 42
  • KF: 2D 상태 [위치, 속도], PID: kp=2.0, kd=2.0, ki=0.5
  • 시뮬: 60초

3구현 과제

closed_loop_kf.py의 closed_loop_step() 함수를 구현하세요. LPF 과제(05)와의 핵심 차이는 prev_u 인자 추가입니다. 

def closed_loop_step(
    plant, estimator, controller, target: float, prev_u: float,
) -> tuple[float, float, float, float]:
    """한 스텝: measure → KF(measure, prev_u) → control → actuate.
    반환: (y_true, y_measure, y_estimate_position, u)"""
인자 의미
plant plant_kf.Plant 인스턴스
estimator KalmanFilter2D 인스턴스
controller PIDController 인스턴스
target 목표값
prev_u 직전 스텝의 제어 입력 (driver loop에서 매 스텝 갱신, 처음엔 0)
반환 (y_true, y_measure, y_estimate_position, u) — 모두 float

호출 순서 (의도):

  1. y_measure = plant.measure()
  2. state = estimator.step(y_measure, prev_u) — prev_u를 두 번째 인자로 전달
  3. y_estimate = float(state[0]) — state[0]은 위치, state[1]은 속도
  4. u = controller.step(target, y_estimate)
  5. plant.step(u)

5합격 기준 pytest

결합 함수의 내부 구현 형태는 자유 — 인터페이스 + 동작 명세(behavioral spec)만 검증합니다.

  1. KF + PID 폐루프 추적 오차 — 외란 0.1, 노이즈 std 0.25, seed 42, 60초 시뮬 후 tail 평균 |y_true| < 0.15, peak < 1.5
  2. KF 추정 정확도 — 추정 위치의 tail 평균 절대 오차 < 0.10 (raw 노이즈 평균 절대값 약 0.20의 절반 이하)
주의: prev_u가 KF에 잘못 전달되거나 누락되면 추정/추적 모두 임계값을 초과합니다. 테스트는 API contract 수준에서 prev_u 전달 여부를 직접 검증합니다.

6힌트

  • KF의 step 시그니처: step(measurement, control_input) — control 입력 누락 시 TypeError가 발생합니다.
  • driver loop에서 prev_u 추적: 처음 0으로 시작, 매 스텝 후 방금 계산된 u로 갱신합니다.
  • state[0]은 numpy scalar — float(state[0])로 명시적 변환이 필요합니다 (테스트의 타입 검증 통과).
  • KF가 LPF보다 위상 지연이 적은 이유는 plant 모델로 측정 전에 상태를 미리 예측하기 때문입니다.
