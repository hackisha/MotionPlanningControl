횡방향 제어 — Kinematic Lateral PID

Chapter 3 · Vehicle Control / 04_kinematic_lateral  |  직선 경로에서 Y 위치를 PID 로 추종하는 첫 횡방향 제어기

자율주행 차량의 횡방향(조향) 제어를 시작합니다. kinematic bicycle 모델에서 PID 로 차량의 Y 위치를 목표 값으로 수렴시키는 제어기를 설계해 보세요.

1문제 상황

차량은 일정 속도 vx = 3 m/s로 X 방향으로 주행하며, Y 방향 위치를 목표 Y_ref = 4 m로 수렴·유지해야 합니다.

  • 제어기의 출력은 조향각 δ (라디안)입니다.
  • 차량 모델이 δ 를 받아 yaw rate 를 만들고, 그 결과로 Y 가 변합니다.
  • 직선 경로 추종 — 곡선 경로는 이후 06/07/08 모듈에서 다룹니다.

2플랜트 모델

vehicle_lat_kinematic.py의 VehicleLat kinematic bicycle 모델입니다. 매 스텝 dynamics: 

δ_clipped  = clip(delta, −0.5, +0.5)        # rad (≈ ±28.6°)
yawrate    = vx / (L + kv·vx²) · δ_clipped
Yaw        ← Yaw + dt · yawrate
X          ← X + vx·dt·cos(Yaw)
Y          ← Y + vx·dt·sin(Yaw)
  • wheelbase L = 4.0 m, 속도 민감 파라미터 kv = 0.005.
  • 조향각이 ±0.5 rad로 포화됩니다 — 큰 값을 출력해도 실제 조향은 제한됩니다.
  • 초기 조건: X=0, Y=0, Yaw=0.

3구현 과제

kinematic_lateral_pid.py의 KinematicLateralPID.step() 메소드를 구현하세요. 

class KinematicLateralPID:
    def __init__(self, kp: float, kd: float, ki: float, dt: float): ...

    def step(self, reference_Y: float, ego_Y: float) -> float
        """조향각 δ (rad) 를 반환."""
인자의미
reference_Y목표 Y 위치 [m]
ego_Y현재 ego 의 Y 위치 [m]
반환조향각 δ [rad] — ego 의 VehicleLat.step() 입력
  • 내부 상태 prev_error, error_sum을 관리합니다.
  • 첫 호출은 D 항 = 0.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py / demo.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — record_gen.py 의 아래 줄. release 기본값은 0 이니 여기에 값을 채우세요:

    pid = KinematicLateralPID(kp=0.0, kd=0.0, ki=0.0, dt=dt)

디버그 신호 — 심화 분석 (선택)

record_gen.py 의 dbg.add(...) 블록에는 debug1/2/3 주석 placeholder 가 준비돼 있습니다. 주석을 풀고 분석하고 싶은 값·식(예: 추적 오차, 내부 상태)을 채우면 그 신호가 record 에 함께 저장됩니다 — 신호 추가·삭제·수정은 이 dbg.add() 호출의 kwarg 한 줄로 끝납니다.

디버그 신호는 viewer 기본 화면에는 나타나지 않습니다 — Rerun viewer 좌측 entity 패널에서 /debug/<이름> 항목을 골라 TimeSeriesView 를 직접 추가하면 시계열로 볼 수 있습니다. 구현이 막힐 때 내부 신호를 들여다보는 디버깅 도구로 활용하세요.

6힌트

  • 조향각 단위는 라디안입니다 — degree 가 아닙니다. 차량 모델이 ±0.5 rad로 클리핑합니다.
  • PID 구조는 01_speed_pid 와 동일 — error 가 속도 대신 Y 위치 차이입니다.
  • D 항(kd)이 중요합니다 — 없으면 차량이 목표선을 지나쳐 진동합니다.
  • 직선 추종은 PID 가 가장 잘하는 시나리오입니다. 곡선 경로는 이후 모듈에서 다룹니다.
